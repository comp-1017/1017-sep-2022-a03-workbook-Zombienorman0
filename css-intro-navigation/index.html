Create Navigational Links Within A Website

There are several techniques to navigate within a website. The semantic tag for navigation is the <nav> tag. You can navigate through the website by using relative addressing. For our examples, we will use the unordered list tag(<ul>).

  • page to page within the same folder
  • page to page between folders
  • staying on the current page
  • going to a specific place on the current page (on page navigation)

Navigational menus are placed in the <header> semantic area. They may also be placed in the <footer> area as a sitemap. the nav tag like other semantic tags should have a <hx> tag. HOWEVER, this header is optional. If omitted the header tag, it will recieve an document outliner error: Untitled Section. Until shown how to avoid the error message, it will be acceptable on the document outline.

CSS Coding Styles Summary

Browser Defaults (no CSS) you have no control over the styling
Inline this is for a single specific element (tag) instance. your css will be specified in a parameter within the element. This css affects ONLY the element tag that contains the specification. style="rule; [rule;...]" <p style="color:purple; width:50px;" > .... <p>
Embedded the specification of css as embedded happens in the <head> area. this css affects all similar elements on the web page where the css is coded. <style>
seletor{rule; [rule; ...]}
[seletor{rule; [rule; ...]} ...]
</style>		 <style>
p {color:purple; width:50px;}
h1 {font-style:italic}
</style>
Linked (StyleSheet) this is the preferrred method of doing css. Place css in a seperate physical file. This file can then be used on ANY web page in the website. The link to the file will be done in the <head> area. The css in the file is similar to embedded EXCEPT there is no style tag. seletor{rule; [rule; ...]}
[seletor{rule; [rule; ...]} ...]
 p {color:purple; width:50px;}
h1 {font-style:italic}
